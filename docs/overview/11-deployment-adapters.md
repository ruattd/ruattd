# Deployment Adapters

astro-koharu 支持自动检测部署平台并选择对应的适配器。

## 适配器选择逻辑

```javascript
/**
 * 按需渲染适配器自动选择
 * 
 * 优先级：
 * 1. 判断是否需要适配器（开发环境看 CMS 是否启用，生产环境始终需要）
 * 2. 检测平台：Vercel > Cloudflare > Netlify
 * 3. 保底方案：Node.js standalone（适合自托管）
 * 4. 无适配器：静态构建
 */
function selectAdapter() {
  const isDev = process.env.NODE_ENV !== 'production';
  const needsAdapter = isDev ? cmsConfig?.enabled : true;
  
  if (!needsAdapter) return undefined; // Static build

  // Platform detection
  const isVercel = process.env.VERCEL === '1';
  const isCloudflare = process.env.CF_PAGES === '1' || !!process.env.CLOUDFLARE;
  const isNetlify = process.env.NETLIFY === 'true';

  if (isVercel) return vercel();
  if (isCloudflare) return cloudflare();
  if (isNetlify) return netlify();
  
  return node({ mode: 'standalone' }); // Fallback
}
```

## 支持的平台

| 平台 | 适配器 | 环境变量检测 |
|------|--------|-------------|
| **Vercel** | `@astrojs/vercel` | `VERCEL=1` |
| **Cloudflare Pages/Workers** | `@astrojs/cloudflare` | `CF_PAGES=1` 或 `CLOUDFLARE` |
| **Netlify** | `@astrojs/netlify` | `NETLIFY=true` |
| **自托管/Docker** | `@astrojs/node` | 其他情况（保底） |

## 部署说明

### Vercel
1. 连接 GitHub 仓库到 Vercel
2. 自动检测并使用 `@astrojs/vercel` 适配器
3. 一键部署：[Deploy with Vercel](https://vercel.com/new/clone?repository-url=https://github.com/cosZone/astro-koharu)

### Cloudflare Pages
1. 连接 GitHub 仓库到 Cloudflare Pages
2. 构建命令：`pnpm build`
3. 输出目录：`dist`
4. 自动使用 `@astrojs/cloudflare` 适配器

### Netlify
1. 连接 GitHub 仓库到 Netlify
2. 构建命令：`pnpm build`
3. 发布目录：`dist`
4. 自动使用 `@astrojs/netlify` 适配器

### 自托管（Node.js）
```bash
# 构建
pnpm build

# 运行
node dist/server/entry.mjs
```

或使用 Docker（见 `/docker` 目录）：
```bash
./docker/rebuild.sh
```

## 本地测试

测试特定平台适配器：

```bash
# Vercel
VERCEL=1 NODE_ENV=production pnpm build

# Cloudflare
CF_PAGES=1 NODE_ENV=production pnpm build

# Netlify
NETLIFY=true NODE_ENV=production pnpm build

# Node.js (默认)
NODE_ENV=production pnpm build
```

## 开发环境

- **CMS 启用**（`config/cms.yaml` 中 `enabled: true`）：使用 Node.js 适配器支持 API 路由
- **CMS 未启用**：纯静态构建，更快的热重载
